# Мульти-расписание: турниры + волны (waves)

## Термины

- **Schedule** — сущность расписания на дату, содержащая:
  - корты (`courts`),
  - запуски/раунды расписания (`runs`),
  - слоты (`slots`),
  - глобальные паузы (`global_breaks`),
  - набор турниров в расписании (`scopes`),
  - группы параллельного старта турниров (`waves`).

- **ScheduleScope** — связь «расписание ↔ турнир».
  - Важно: в мульти-расписании **один Schedule** обслуживает **несколько турниров**.

- **ScheduleWave** — «волна» (группа старта): несколько турниров могут стартовать параллельно.
  - Волны имеют порядок (`order`) и режим старта.

- **Bound courts / привязанные корты** — привязка кортов к конкретному турниру в рамках расписания.
  - Хранится в `ScheduleScopeCourt`.

## Модель данных (backend)

- `apps/schedules/models.py`
  - `ScheduleWave(schedule, order, start_mode, start_time, earliest_time)`
  - `ScheduleScope(schedule, wave, tournament, order, start_mode, start_time)`
  - `ScheduleScopeCourt(scope, court)`

### Start modes

- Для **волны** (`ScheduleWave.start_mode`):
  - `fixed` — волна стартует не раньше указанного `start_time`.
  - `after_previous` — волна может стартовать после предыдущей по порядку, но ограничивается `earliest_time` (если задано).

- Для **scope** (`ScheduleScope.start_mode`) — override поведения старта для конкретного турнира.
  - В UI это спрятано в «Дополнительно».

## API (backend)

### Расписание (Schedule)

- `POST /api/schedules/<id>/save/`
  - Сохраняет корты/запуски/слоты/паузы.
  - Важно: корты и запуски пересоздаются, поэтому при сохранении требуется аккуратно сохранять связанный с ними state.

### Scopes (турниры в расписании)

- `POST /api/schedules/<id>/scopes/add/`
- `POST /api/schedules/<id>/scopes/remove/`
- `POST /api/schedules/<id>/scopes/reorder/`
- `POST /api/schedules/<id>/scopes/update/`
  - В том числе принимает `bound_courts: number[]`.
- `POST /api/schedules/<id>/scopes/move_to_wave/`

### Waves (волны)

- `POST /api/schedules/<id>/waves/add/`
- `POST /api/schedules/<id>/waves/update/`
- `POST /api/schedules/<id>/waves/reorder/`
- `POST /api/schedules/<id>/waves/remove/`

## UI (frontend)

Основная страница управления расписанием: `frontend/src/pages/SchedulePage.tsx`.

### Модалка «Турниры в расписании»

Позволяет:
- создавать/удалять/переставлять **волны**,
- добавлять/удалять/переставлять **турниры (scopes)**,
- переносить турнир между волнами,
- редактировать параметры волны и (опционально) override старта конкретного турнира.

## Авторасстановка

Логика авторасстановки находится в `SchedulePage.tsx` в `autoAssignAndSave()`.

Основные ограничения:
- **Волны**:
  - турниры в одной волне считаются параллельными;
  - следующая волна подключается, когда появляется возможность ставить матчи в поздних `runs`, но не раньше ограничений по времени (`fixed start_time` / `earliest_time`).
- **Конфликты игроков**:
  - запрещено ставить матчи, если игрок уже занят в том же `run` (в рамках всех турниров расписания).
- **Привязка кортов**:
  - если у турнира есть `bound_courts`, то его матчи можно ставить только на эти корты;
  - если у каких-то турниров есть `bound_courts`, то остальные турниры считаются ограниченными «оставшимися» кортами.
- **KO (олимпийка) prereqs**:
  - матчи следующего раунда не ставятся раньше матчей предыдущего раунда, от которых они зависят.

## Проверочные сценарии (smoke)

- Создать расписание для турнира A.
- Добавить турнир B в волну 1, турнир C в волну 2.
- Нажать «Очистить» → «Авто» и убедиться, что:
  - матчи волны 1 начинают заполняться раньше волны 2,
  - волна 2 не стартует раньше `earliest_time` / `fixed start_time`.
- Для KO турнира убедиться, что:
  - матчи раунда 1 не оказываются в run’ах раньше матчей раунда 0.
